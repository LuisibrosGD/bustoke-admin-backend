PlaintextActúa como un Ingeniero DevOps / DevSecOps. Necesito integrar la suite de pruebas automatizadas que acabamos de definir en un flujo de Integración Continua (CI/CD) utilizando GitHub Actions. 

Por favor, escribe el archivo de configuración .github/workflows/test-automation.yml completo que realice las siguientes acciones en cada 'push' o 'pull_request' a la rama principal:
1. Levantar el entorno necesario (instalar dependencias de Node.js/Python, configurar variables de entorno esenciales).
2. Levantar la aplicación o servicios en background (si aplica).
3. Ejecutar de forma automática la suite de pruebas que definimos en el paso anterior (especifica el comando de ejecución).
4. Exportar o guardar los reportes de resultados de las pruebas como artefactos del workflow.

Asegúrate de que la sintaxis sea completamente válida para GitHub Actions y esté optimizada para un proyecto que usa la arquitectura actual de nuestro repositorio.
